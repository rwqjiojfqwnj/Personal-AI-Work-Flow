# 常用提示词模板

> 用途：给 AI 提供稳定、可复制的工作流入口，减少每次重新组织语言。

---

## 1. 项目初始化

```text
这是一个前后端分离项目：
- 前端：Vue 3 + TypeScript + Element Plus
- 后端：Spring Boot + Java
- 契约源：OpenAPI

请按项目基线初始化工作流：
1. 检查 comet / openspec / superpowers / codegraph 是否已就绪
2. 建立或检查 docs/workflow-templates/
3. 建立项目知识图谱
4. 确认前后端基础项目可启动
5. 输出初始化结果和后续建议
```

---

## 2. 需求澄清与立项

```text
我正在开发一个新功能：[一句话描述]

输入材料：
- PRD：docs/prd/xxx.md
- 外部接口文档：docs/api-external/xxx.md
- 现有项目代码结构：请参考 codegraph 图谱

请按 /comet-open 的思路完成：
1. 识别业务目标、本期边界、非本期范围
2. 标出关键风险、外部依赖、待确认问题
3. 输出 proposal、初版 spec、tasks 建议
4. 给出验收关注点
```

---

## 3. 深度设计与契约冻结

```text
需求已澄清，请进入深度设计阶段。

要求：
1. 基于 OpenAPI 作为单一契约源进行设计
2. 先做统一设计，再做角色化设计
3. 明确前端页面、后端接口、权限、异常、异步任务
4. 输出验收矩阵初版
5. 判断是否适合多 Agent 并行
```

---

## 4. 前端页面设计与样式

```text
请设计并实现这个页面：[页面名称]

约束：
- 组件库固定为 Element Plus
- 只做业务系统风格，不做营销型视觉探索
- 样式应优先复用组件能力，再补页面级布局和细节
- 需要包含 loading、空状态、错误状态、权限状态

完成后请：
1. 自查页面结构和交互
2. 给出是否建议执行 /impeccable critique 或 /impeccable polish
```

---

## 5. 角色化设计拆分

```text
统一设计已确认，请继续按角色化视角拆分：
1. 产品/业务视角
2. 后端设计视角
3. 前端设计视角
4. 测试/验收视角

输出时请明确：
- 每个视角最关键的设计决策
- 各自的风险点
- 适合由哪个 Agent 承担
```

---

## 6. 构建与多 Agent 协作

```text
设计已冻结，请进入构建阶段。

要求：
1. 先判断适合单 Agent、双 Agent 还是三 Agent
2. 若并行，明确 frontend-agent / backend-agent / qa-agent 的边界
3. 以 OpenAPI 为唯一契约事实源
4. 前端默认基于 Element Plus 开发
5. 输出联调清单和阶段完成标准
```

---

## 7. 验证与验收

```text
开发已完成，请执行验证与验收。

要求：
1. 检查 OpenAPI 与前端 types、后端 DTO 是否一致
2. 检查测试、构建、联调结果
3. 按验收矩阵逐项确认
4. 对重点页面建议是否执行 /impeccable audit
5. 输出验证报告和遗留问题
```
