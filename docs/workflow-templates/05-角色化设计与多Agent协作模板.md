# 角色化设计与多 Agent 协作模板

> 用途：在统一设计完成后，把需求拆成不同角色视角，并决定是否允许并行执行。

---

## 1. 统一设计前提

只有满足以下条件，才进入角色化设计：

- 范围清晰
- 风险已识别
- 单一契约源已明确
- 初版验收标准已存在

## 5. 并行前必须冻结的内容

- 统一设计文档
- 单一契约源

- 验收矩阵初版

---

## 6. 禁止并行条件

- 需求仍在持续变化
- 接口仍在争议中
- 关键领域模型未定
- 三方接口可行性未确认

---

## 7. 同步节奏建议

- 设计冻结后同步一次
- API 骨架完成后同步一次
- 联调前同步一次
- 验收前同步一次
