# 前端页面开发模板

> 用途：用于 Vue 3 + TypeScript + Element Plus 项目中的页面级开发。  
> 目标：让 AI 在企业中后台场景下稳定输出可联调、可验收、可维护的页面实现。

---

## 1. 适用场景

- 列表页
- 搜索页
- 详情页
- 新增 / 编辑页
- 弹窗表单页
- 组合型业务页

---

## 2. 固定前提

- 技术栈：`Vue 3 + TypeScript + Element Plus`
- 契约事实源：`OpenAPI`
- 页面样式：优先复用 `Element Plus`
- 只做业务系统风格，不做营销型视觉探索

---

## 3. 开发前必须确认

1. 页面对应哪个需求点
2. 页面对应哪些接口
3. 是否已有现成组件可复用
4. 页面是否涉及权限控制
5. 页面是否有空状态、错误状态、加载状态
6. 页面是否需要草稿、离开确认、重复提交防护

## 5. 推荐提示词模板

```text
请实现一个前端页面：[页面名称]

项目约束：
- 技术栈：Vue 3 + TypeScript + Element Plus
- 接口契约以 OpenAPI 为准
- 优先复用已有公共组件和页面模式
- 样式以业务系统风格为主，不做营销型视觉探索

请按以下要求输出并实现：
1. 先说明页面结构、组件拆分和状态设计
2. 明确对应的接口与数据结构
3. 优先使用 Element Plus 组件实现
4. 必须包含 loading、空状态、错误状态、权限状态
5. 表单提交要防重复点击
6. 如涉及未保存内容，补离开确认
7. 完成后说明是否建议执行 /impeccable critique 或 /impeccable audit
```

---

## 6. 实现检查清单

- 是否优先复用了 Element Plus 组件
- 是否优先复用了已有业务组件
- 是否避免了页面逻辑和样式过度耦合
- 是否包含 loading / 空 / 错误 / 权限状态
- 是否避免硬编码接口地址
- 是否与 OpenAPI 契约一致
- 是否便于联调和验收

---

## 7. Impeccable 使用建议

### 建议使用的时机

- 页面首次落地前：`/impeccable shape`
- 页面初版完成后：`/impeccable critique`
- 提测前：`/impeccable audit`
- 交付前：`/impeccable polish`

### 不建议依赖 Impeccable 的场景

- 纯接口联调阶段
- 只是修一个字段展示
- 完全已有统一页面骨架的小改动

---

## 8. 联调前自检

- 接口参数是否与 OpenAPI 一致
- 列表分页、筛选、排序是否完整
- 表单校验是否完整
- 操作成功 / 失败提示是否明确
- 页面是否存在明显样式错位
