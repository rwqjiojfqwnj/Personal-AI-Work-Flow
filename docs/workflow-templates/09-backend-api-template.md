# 后端接口开发模板

> 用途：用于 Java / Spring Boot 前后端分离项目中的接口级开发。  
> 目标：让 AI 在企业后端开发中稳定输出契约一致、异常明确、易联调的接口实现。

---

## 1. 适用场景

- CRUD 接口
- 分页查询接口
- 详情接口
- 表单提交接口
- 状态流转接口
- 三方接口适配层

---

## 2. 固定前提

- 技术栈：`Spring Boot + Java`
- 接口契约事实源：`OpenAPI`
- 前后端分离
- 返回结构、异常结构、权限策略遵守项目基线

---

## 3. 开发前必须确认

1. 该接口对应哪个需求点
2. OpenAPI 是否已定义完成
3. 输入输出字段是否冻结
4. 是否涉及权限、幂等、事务
5. 是否涉及缓存、异步任务、三方接口
6. 是否有明确错误码和失败场景

## 5. 推荐提示词模板

```text
请实现一个后端接口：[接口名称]

项目约束：
- 技术栈：Spring Boot + Java
- 接口契约以 OpenAPI 为准
- 返回结构、异常处理、分页结构遵守项目基线

请按以下要求输出并实现：
1. 先说明接口职责、输入输出、异常场景
2. 对照 OpenAPI 检查字段是否一致
3. 明确 Controller / Service / Repository(Mapper) 的职责边界
4. 参数必须做校验
5. 明确成功、失败、无权限、重复提交等场景
6. 如涉及三方接口，说明超时、重试、日志记录策略
7. 完成后给出联调注意点和测试建议
```

---

## 6. 实现检查清单

- 是否严格对齐 OpenAPI
- 是否存在未说明的字段漂移
- 是否有统一参数校验
- 是否有明确异常处理
- 是否避免循环 SQL
- 是否避免大字段进入列表查询
- 是否对分页、筛选、排序做了统一约束
- 是否输出了联调和测试关注点

---

## 7. 三方接口场景补充

若该接口依赖三方系统，至少补充：

- 超时策略
- 重试策略
- 降级策略
- 请求 / 响应日志
- 敏感信息脱敏
- 错误码映射

---

## 8. 联调前自检

- OpenAPI 是否已更新
- DTO / VO 是否与契约一致
- 前端关心的字段是否齐全
- 错误提示是否可被前端消费
- 日志是否足够排查问题
